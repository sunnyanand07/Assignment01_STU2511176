## Architecture Recommendation

For a fast-growing food delivery startup, I would recommend using a
Data Lakehouse architecture. This approach combines the flexibility of a
Data Lake with the performance and structure of a Data Warehouse.

Firstly, the startup collects diverse types of data such as GPS logs,
text reviews, transactions, and images. A Data Lake is ideal for storing
raw, unstructured, and semi-structured data in its native format without
requiring upfront schema design.

Secondly, the Data Lakehouse allows structured querying and analytics
on top of this raw data. This is useful for generating business insights,
such as delivery performance, customer behavior, and revenue trends.

Thirdly, scalability is crucial for a growing startup. A Data Lakehouse
can handle large volumes of data efficiently while supporting both batch
processing and real-time analytics.

Thus, a Data Lakehouse provides the best balance between flexibility,
scalability, and analytical performance, making it the most suitable
choice for this scenario.
